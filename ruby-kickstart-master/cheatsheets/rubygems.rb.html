1# =====  What Are Gems?  =====
 2# In Ruby, third party libraries are called gems
 3# You can see a list of most of Ruby's gems at rubygems.org
 4
 5
 6# =====  How To Install  =====
 7# When you find one there that you are interested in, e.g. BlueCloth, a gem to parse markdown and output html
 8# http://rubygems.org/gems/bluecloth
 9# It will give you instructions on how to install it
10# $ gem install bluecloth
11#
12# (note that if you are on OSX, and not using rvm, and haven't fixed your path, you may need to execute that with sudo)
13
14
15# =====  What Happens When You Install?  =====
16# When you install a gem, it, in turn, installs its dependencies
17# So installing Sinatra will also install rack
18# BlueCloth doesn't have any dependencies 
19
20# It also builds code that is written in a language that needs to be compiled (namely C)
21# So some libraries, like BlueCloth will require you to have a compiler available to use to compile the gem for your system
22
23# It also builds documentation in the form of ri, and rdoc
24# To use ri, go to the command line and type "$ ri topic", ie "$ ri String"
25# To use the rdocs, you can load up your own documentation server
26
27
28# =====  Documentation  =====
29# To load up your own documentation Server
30# $ gem server
31#
32# It will be hosted on some local port, probably 8808
33# Go to http://localhost:8808
34# Select the gem of interest, the docs are there (for example, my bluecloth is at)
35

37# =====  Where Else To Look When Figuring Out How To Use It  =====
38# I find a combination of resources helpful, docs are just one
39#
40# Looking through a gem's source code to see how they implement
41# A given method, or how the magic works, can be very helpful
42#
43# To get there, you you can go to your gem directory, see this with
44# $ gem environment
45# 
46# If you have gem-open ("$ gem install gem-open"), then you can open it directly with "$ gem open gemname"
47# 
48# If you just want to see it quickly, you can follow the link to the source host (probably github)
49# If you want to experiment with it, you can git clone the source code into a directory on your machine
50# Then you can play around without affecting your gems.
51#
52# A good example can also be very helpful, google + blogs can be your friend :)
53# Oftentimes there are also examples in the code
54#
55# Looking at what things the authors taken the time to test can show you how they expected it to be used
56# And what features and things they took the time to make sure work correctly
57# Tests will either be in /spec or /test directories.
58#
59# For example, looking at the BlueCloth homepage I didn't see anything helpful.
60# I loaded up the gem server, and felt unsure where the "overview" information was
61# I looked at spec/bluecloth_spec.rb and saw the spec
62# it "allows output to be rendered several times" do
63#   bc = BlueCloth.new( "Some text" )
64#   bc.to_html.should == bc.to_html
65# end
66#
67# Oh, I guess we can just drop our markdown into the constructor, and call the instance method to_html (see me do it below)
68

70# =====  Using Gems  =====
71# Gems become available to you in your ruby files
72# (On Ruby 1.8, you must require rubygems first.)
73require 'bluecloth'
74
75# now you are free to use it as you would any library, or even your own code
76topics = BlueCloth.new <<CHAPTER_DESCRIPTION
77* Arithmetic
78* Logic
79* Basic IO
80* Arrays
81CHAPTER_DESCRIPTION
82
83topics.to_html # => "<ul>\n<li>Arithmetic</li>\n<li>Logic</li>\n<li>Basic IO</li>\n<li>Arrays</li>\n</ul>"
